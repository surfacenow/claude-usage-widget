# Live Design Panel 指示書

## 概要

`claude_overlay.py` の右クリックメニューに「Design Mode」を追加。
クリックすると調整パネルウィンドウが開き、マウス操作でウィジェットの見た目をリアルタイム調整できる。

## 目的

「コード変更 → 再起動 → スクショ → 言葉で説明」のループを排除する。
全ての見た目調整をマウス操作 + 即時反映で完結させる。

---

## パネル構成

1つのToplevelウィンドウ。セクションごとにLabelFrameで区切る。

### セクション1: Layout（レイアウト）

| パラメータ | コントロール | 範囲 | 対応する変数 |
|-----------|------------|------|-------------|
| グリップ幅 | スライダー | 8–30 | `grip_w` |
| ラベルX | スライダー | 10–40 | `lbl_x` |
| バーX | スライダー | 20–60 | `bar_x` |
| バー幅 | スライダー | 30–120 | `bar_w` |
| %X | スライダー | 60–180 | `pct_x` |
| リセットX | スライダー | 80–200 | `rst_x` |
| 行高さ | スライダー | 8–20 | `row_h` |
| Y0 オフセット | スライダー | 0–10 | `y0` |

### セクション2: Grip（グリップドット）

| パラメータ | コントロール | 範囲 | 対応する変数 |
|-----------|------------|------|-------------|
| ドット半径 | スライダー(0.1刻み) | 1.0–5.0 | `dot_r` |
| 列1 X | スライダー | 2–16 | `dx[0]` |
| 列2 X | スライダー | 2–16 | `dx[1]` |
| 行1 Y | スライダー | 2–30 | `dy[0]` |
| 行2 Y | スライダー | 2–30 | `dy[1]` |
| 行3 Y | スライダー | 2–30 | `dy[2]` |

### セクション3: Colors（配色）

現在選択中のテーマの色を編集する。各色にボタンを置き、クリックで `colorchooser.askcolor()` を開く。

| パラメータ | 対応するテーマキー |
|-----------|------------------|
| アクセント | `accent` |
| ラベル文字 | `muted` |
| バー背景 | `bar_bg` |
| バー(低) | `bar_low` |
| バー(中) | `bar_mid` |
| バー(高) | `bar_high` |
| テキスト(低) | `usage_low` |
| テキスト(中) | `usage_mid` |
| テキスト(高) | `usage_high` |
| リセット文字 | `reset_text` |
| リセット影 | `reset_shadow` |

各ボタンはその色で塗りつぶし表示。押すとカラーピッカーが開き、選んだ色が即反映される。

### セクション4: Font（フォント）

| パラメータ | コントロール | 範囲 |
|-----------|------------|------|
| メインフォントサイズ | スライダー | -8 – -18 |
| リセットフォントサイズ | スライダー | -8 – -16 |
| Bold ON/OFF | チェックボックス | — |

## 操作フロー

```
右クリック → 「Design Mode」
    ↓
パネルウィンドウが開く（ウィジェットの近くに配置）
    ↓
スライダーやカラーピッカーで調整
    ↓
変更は即座にウィジェット本体に反映（command/trace コールバック）
    ↓
「Export」ボタン → 現在の値を design_values.json に書き出し
「Reset」ボタン → コード上のデフォルト値に戻す
「Close」ボタン → パネルを閉じる
```

## 技術仕様

### パネルウィンドウ
- `tk.Toplevel` で生成
- `overrideredirect` は使わない（通常ウィンドウとして扱う）
- ウィジェットの左側に配置（画面端を考慮）
- パネルを閉じてもウィジェットは動き続ける

### リアルタイム反映
- スライダー: `command=` コールバックで即座に `place()` / `config()` を再実行
- カラーピッカー: `colorchooser.askcolor()` の戻り値で即 `config()`
- グリップドット: `coords()` で既存のovalの座標を更新（再生成不要）

### Export
- `design_values.json` に以下の形式で書き出し:

```json
{
  "layout": {
    "grip_w": 18,
    "lbl_x": 20,
    "bar_x": 38,
    "bar_w": 76,
    "pct_x": 116,
    "rst_x": 152,
    "row_h": 12,
    "y0": 2
  },
  "grip": {
    "dot_r": 2.0,
    "dx": [5, 13],
    "dy": [7, 14, 21]
  },
  "font": {
    "main_size": -12,
    "reset_size": -11,
    "bold": false
  },
  "themes": {
    "dark_bg": { ... },
    "light_bg": { ... }
  }
}
```

- このJSONがあれば次回のClaude Code作業で「この値をコードに反映して」と渡すだけで済む
- 将来的にはウィジェット起動時にこのJSONから読み込む拡張も可能

## 実装上の注意

- Design Modeはデバッグ/調整用。通常ユーザーには見せない想定
- メニューの表示条件として `--design` 起動引数か、configの `"design_mode": true` で制御する
- パネルのスライダーが多いのでスクロール可能にする（Canvas + Scrollbar）
- ウィジェットサイズ (`w`, `h`) もレイアウト変更に応じて自動拡縮するか、手動スライダーにするか → 手動スライダー推奨
